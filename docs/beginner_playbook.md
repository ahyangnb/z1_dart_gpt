# 新手完整指南：怎么玩这个 Dart 小 GPT，以及它和真实 AI 开发的关系

这份文档面向刚开始接触 AI、机器学习、GPT 或 Dart 的读者。目标不是让你背术语，而是把这个项目拆成一个可以玩的、可以观察的、可以改动的学习实验。

读完后你应该能回答这些问题：

- 这个项目到底是什么？
- 怎么运行？怎么改？怎么玩出不同结果？
- 它为什么能生成“像名字”的字符串？
- 常规人工智能项目有哪些流程？
- 这个项目里的每段代码负责 AI 流程中的哪一步？
- 如果以后要做完整商业级 AI 产品，可以从这里怎么升级？

## 1. 一句话理解这个项目

`z1_dart_gpt.dart` 是一个“极小版 GPT 训练器”。

它会读取一堆中文名字，然后学习：

```text
看到前面的字符后，下一个字符最可能是什么？
```

例如训练数据里有很多名字：

```text
王伟
李若然
张明宇
刘雨桐
```

模型慢慢会发现：

- 开头位置经常是常见姓氏，比如 `王`、`李`、`张`、`刘`。
- 姓氏后面经常接名字用字，比如 `若`、`明`、`雨`、`梓`。
- `若然`、`明宇`、`雨桐` 这类组合很像中文名。
- 一个名字生成到合适长度后应该停止。

训练完成后，它可以从一个开始标记出发，一个字符一个字符地生成新名字。

这就是 GPT 的核心思想：

```text
根据上下文预测下一个 token。
```

大型聊天模型也是这个思想，只是它们的 token 更复杂、参数更多、数据更大、工程系统更完整。

## 2. 你在玩什么

你不是在玩一个普通随机名字生成器。

普通随机生成器可能写死一些规则，例如：

```text
随机选 1 个姓氏 + 2 个名字用字
```

这个项目不是这样。它没有写死“名字应该长什么样”。它是通过训练数据自己学出来的。

训练前：

```text
模型参数是随机数，输出也基本乱。
```

训练后：

```text
模型参数被 loss 和梯度一点点调整，输出开始像名字。
```

所以你玩的其实是一个完整的 AI 学习过程，只是尺寸非常小。

## 3. 怎么运行

在项目根目录执行：

```bash
dart run bin/z1_dart_gpt.dart
```

运行前需要在项目根目录准备本地 `input.txt`。这个文件里每一行是一个名字。程序只加载本地文件，不会联网下载。

可以先创建一个很小的 `input.txt` 试跑：

```text
王伟
李若然
张明宇
刘雨桐
陈梓涵
```

你会看到类似输出：

```text
num docs: 44928
vocab size: 328
num params: 13824
step 1000 / 1000 | loss 3.2432
--- 推理结果（生成的中文名字）---
样本  1: 韦若然
样本  2: 苏子然
样本  3: 曹明宇
...
```

这些输出分别表示：

| 输出           | 含义                           |
|--------------|------------------------------|
| `num docs`   | 训练样本数量，也就是名字数量               |
| `vocab size` | token 数量，这里是中文字符加 1 个 BOS    |
| `num params` | 模型里可训练参数的数量                  |
| `loss`       | 模型当前预测错误程度，越低通常越好            |
| `样本`         | 训练后模型生成的新名字                  |

## 4. 怎么快速试玩

默认训练 1000 步，可能需要等一会儿。你可以先用更少步数观察变化：

```bash
Z1_DART_GPT_STEPS=1 dart run bin/z1_dart_gpt.dart
Z1_DART_GPT_STEPS=10 dart run bin/z1_dart_gpt.dart
Z1_DART_GPT_STEPS=100 dart run bin/z1_dart_gpt.dart
Z1_DART_GPT_STEPS=1000 dart run bin/z1_dart_gpt.dart
```

你可以观察：

- 训练步数很少时，生成结果比较乱。
- 训练步数增加后，结果更像名字。
- loss 通常会逐渐降低，但不是每一步都严格下降。

为什么 loss 不一定每一步都下降？

因为这个项目每一步只拿一个名字训练，不是一次拿很多样本做 batch。单个样本可能比较容易，也可能比较难，所以 loss 会波动。真实训练中也经常看到 loss 有抖动。

## 5. 怎么改数据集

`input.txt` 是训练数据。你可以自己改它。

例如改成水果名：

```text
苹果
香蕉
橙子
西瓜
葡萄
柠檬
```

再运行：

```bash
dart run bin/z1_dart_gpt.dart
```

模型就会尝试生成“像水果名”的字符串。

也可以改成：

- 公司名
- 游戏角色名
- 城市名
- 产品代号
- 武侠角色名
- 你自己的昵称列表

注意：数据越少，模型越容易过拟合，生成结果可能只是记住原数据或变得很怪。

## 6. 怎么改模型大小

代码顶部有几个超参数：

```dart
const nLayer = 1;
const nEmbd = 16;
const blockSize = 16;
const nHead = 4;
const headDim = nEmbd ~/ nHead;
```

它们决定模型大小和上下文能力。

| 参数          | 可以怎么玩    | 影响                   |
|-------------|----------|----------------------|
| `nLayer`    | 改成 2 或 3 | 层数更多，表达能力更强，但更慢      |
| `nEmbd`     | 改成 32    | 向量更宽，参数更多，也更慢        |
| `blockSize` | 改成 32    | 可以处理更长名字             |
| `nHead`     | 改成 2、4、8 | 注意力头数量，必须能整除 `nEmbd` |

建议新手这样试：

```dart
const nLayer = 1;
const nEmbd = 32;
const blockSize = 16;
const nHead = 4;
```

如果你把模型改大，运行会更慢，因为这个项目是标量级教学实现，没有 GPU 加速。

## 7. 怎么改生成风格

推理阶段有一个参数：

```dart
const temperature = 0.5;
```

它在 `runInference()` 里。

temperature 控制生成的随机程度。

| temperature | 效果            |
|-------------|---------------|
| `0.2`       | 更保守，更容易生成常见模式 |
| `0.5`       | 当前默认值，比较稳     |
| `1.0`       | 更多样，也更容易怪     |
| `1.5`       | 很随机，可能不像名字    |

你可以改成：

```dart
const temperature = 1.0;
```

再运行看看生成结果有什么变化。

## 8. 为什么每次运行结果一样

代码里有：

```dart
final rng = math.Random(42);
```

这表示固定随机种子。

固定种子的好处是：

- 方便调试
- 方便复现实验
- 方便比较修改前后的效果

如果你想每次都不一样，可以改成：

```dart
final rng = math.Random();
```

但学习阶段建议先保留固定种子，这样你更容易判断“是代码改动导致变化”，还是“随机性导致变化”。

## 9. 这个东西的核心原理

核心原理可以压缩成一句话：

```text
用大量样本训练一个函数，让它根据已有 token 预测下一个 token。
```

在这个项目里：

```text
输入：前面的字符
输出：下一个字符的概率
```

比如名字 `王伟` 会被变成：

```text
[BOS, 王, 伟, BOS]
```

训练时模型会学习这些任务：

```text
看到 BOS，预测 王
看到 王，预测 伟
看到 伟，预测 BOS
```

每次模型预测错了，就会产生 loss。反向传播会计算每个参数对 loss 的影响。Adam 优化器会根据这些梯度调整参数。

重复很多次后，模型参数就从随机数变成了“保存名字规律的数字”。

## 10. 常规 AI 项目的完整流程

一个常规 AI 项目通常不是从写模型开始，而是从问题开始。

完整流程一般是：

1. 定义问题：要解决什么业务问题？
2. 收集数据：有哪些样本可以学习？
3. 清洗数据：去掉错误、重复、脏数据。
4. 数据编码：把文字、图片、音频转成模型能处理的数字。
5. 选择模型：用传统机器学习、CNN、RNN、Transformer，还是直接用已有大模型？
6. 训练模型：让模型在数据上反复学习。
7. 评估模型：判断效果是否达标。
8. 推理部署：把模型放到 API、App 或服务里使用。
9. 监控反馈：观察线上效果、成本、延迟、错误。
10. 持续迭代：补数据、调模型、改产品体验。

这个项目把这些流程压缩到了一个文件里。

## 12. 和真实 GPT 的共同点

这个项目虽然小，但和真实 GPT 有很多共同点：

- 都是根据上下文预测下一个 token。
- 都需要 tokenizer。
- 都有 embedding。
- 都有 attention。
- 都有 MLP。
- 都有 residual connection。
- 都通过 loss 衡量错误。
- 都通过 backward 计算梯度。
- 都通过 optimizer 更新参数。
- 都能在训练后进行 autoregressive generation。

`autoregressive generation` 的意思是：

```text
先生成第 1 个 token，再把第 1 个 token 放回上下文生成第 2 个 token，如此循环。
```

聊天模型生成回答也是这样做的。

## 13. 和真实商业级 AI 的区别

这个项目是教学版，所以很多地方故意简单。

| 本项目           | 商业级 AI 系统                         |
|---------------|-----------------------------------|
| 一个 Dart 文件    | 多服务、多模块工程                         |
| 字符级 tokenizer | BPE、SentencePiece 或模型专用 tokenizer |
| 约 13824 个参数    | 数百万、数十亿甚至更多参数                     |
| 标量自动求导        | 张量自动求导                            |
| CPU 单线程       | GPU/TPU/分布式训练                     |
| 一次训练一个名字      | batch + data loader               |
| 没有验证集         | train/validation/test 分离          |
| 只打印 loss      | 完整指标、评估集、可视化                      |
| 无模型保存         | checkpoint、版本管理、回滚                |
| 命令行生成         | API、Web、App、SDK                   |
| 无安全治理         | 权限、审计、风控、内容安全                     |

所以不要把它当成生产模型。它更像一台透明的教学发动机：小，但能看清每个齿轮怎么动。

## 14. 如果要做商业级版本，可以怎么演进

下面是从这个教学项目走向完整商业级 AI 系统的思路。

### 14.1 数据层升级

现在：

```text
input.txt，一行一个名字。
```

商业级可以改成：

- 使用数据库或对象存储保存数据。
- 建立训练集、验证集、测试集。
- 增加数据清洗脚本。
- 记录数据来源、授权、版本。
- 做隐私处理和敏感信息过滤。

代码思路：

```text
input.txt -> data/raw/*.jsonl -> data/processed/*.jsonl -> dataset version
```

### 14.2 Tokenizer 升级

现在：

```text
字符级 tokenizer。
```

商业级可以改成：

- BPE tokenizer
- WordPiece tokenizer
- SentencePiece tokenizer
- 直接复用成熟模型 tokenizer

为什么要升级？

字符级 tokenizer 太细，序列会变长。真实文本里有中文、英文、数字、符号、emoji、代码，字符级会很低效。

升级后：

```text
王明宇 -> [王, 明宇]
```

而不是：

```text
王 明 宇
```

### 14.3 自动求导升级

现在：

```text
Value 是标量自动求导。
```

商业级可以改成：

- PyTorch
- TensorFlow
- JAX
- ONNX Runtime
- llama.cpp / ggml 类推理框架

为什么要升级？

标量自动求导适合学习，但非常慢。真实模型需要张量计算，把大量数字一次性并行处理。

### 14.4 模型结构升级

现在：

```text
1 层，小 embedding，ReLU，简单 RMSNorm。
```

商业级可以考虑：

- 更多 Transformer 层
- 更大的 hidden size
- RoPE 位置编码
- SwiGLU / GeLU 激活
- 更标准的 LayerNorm / RMSNorm
- FlashAttention
- KV cache
- dropout
- 权重 tying

如果目标不是从零训练大模型，也可以直接基于开源模型：

- Llama
- Qwen
- Mistral
- Gemma
- DeepSeek 系列

对于商业产品，更多时候不是从零训练，而是：

```text
选基础模型 -> RAG -> 微调 -> 评估 -> 部署
```

### 14.5 训练系统升级

现在：

```text
单文件、单线程、一次一个样本。
```

商业级可以改成：

- batch training
- data loader
- checkpoint 保存
- experiment tracking
- GPU 训练
- mixed precision
- distributed training
- learning rate schedule
- gradient clipping
- resume training

训练系统要解决的不只是“能不能学”，还要解决：

- 训练失败能不能恢复？
- 哪个实验效果最好？
- 训练花了多少钱？
- 数据和模型版本是否可追溯？

### 14.6 评估系统升级

现在：

```text
只看训练 loss 和生成样例。
```

商业级必须有评估。

可以增加：

- validation loss
- test loss
- 自动评测指标
- 人工评测
- A/B 测试
- 失败案例收集
- 回归测试集

对于名字生成器，评估可以包括：

- 是否像名字
- 是否重复训练集
- 是否长度合理
- 是否包含非法字符
- 用户是否喜欢

对于聊天机器人，评估还会包括：

- 准确性
- 有用性
- 幻觉率
- 安全性
- 拒答质量
- 延迟和成本

### 14.7 推理服务升级

现在：

```text
命令行里直接生成 20 个中文名字样本。
```

商业级可以改成：

- HTTP API
- WebSocket 流式输出
- 移动端 SDK
- Web UI
- 批量请求
- 缓存
- 限流
- 权限认证
- 日志和监控

推理服务要关注：

- 响应速度
- 并发能力
- 成本
- 稳定性
- 安全策略

### 14.8 产品层升级

现在：

```text
只是命令行演示。
```

商业级产品需要回答：

- 用户是谁？
- 用户为什么需要它？
- 生成结果如何编辑、保存、分享？
- 失败时怎么提示？
- 是否需要账号、额度、付费？
- 是否有团队协作、历史记录、审计？

例如如果做“AI 品牌命名工具”，产品可以包括：

- 输入行业、风格、长度
- 生成多个候选名
- 检查重复和敏感词
- 检查域名可用性
- 给出含义解释
- 收藏和导出

### 14.9 安全与合规升级

现在：

```text
没有安全系统。
```

商业级必须考虑：

- 用户输入是否包含隐私
- 输出是否有敏感内容
- 日志是否泄露数据
- 数据是否有版权风险
- 是否需要内容审核
- 是否需要权限隔离
- 是否满足所在地区法规

对于企业 AI，安全和合规不是附加项，而是上线条件。

## 15. 如果你想继续练手，可以做这些小任务

从简单到稍难：

1. 把 `temperature` 改成 `0.2`、`1.0`、`1.5`，比较输出。
2. 把 `Z1_DART_GPT_STEPS` 分别设成 `1`、`10`、`100`、`1000`，观察 loss。
3. 自己写一个 `input.txt`，训练水果名、城市名或产品名。
4. 把 `nEmbd` 从 `16` 改成 `32`，观察参数数量和速度。
5. 增加 validation 数据，训练时同时打印验证 loss。
6. 给模型保存参数，下次启动时加载参数继续生成。
7. 写一个简单 Web API，让浏览器请求生成名字。
8. 把字符级 tokenizer 改成更大的 token 单元。
9. 把标量 `Value` 改成向量或矩阵计算。
10. 用成熟框架复刻同样模型，对比速度和代码结构。

## 16. 最重要的心智模型

初学 AI 时，不要先把 GPT 想成一个神秘黑盒。可以先这样想：

```text
模型 = 一个有很多可调数字的函数
训练 = 用数据不断调整这些数字
loss = 告诉模型现在错得多不多
梯度 = 告诉每个数字应该往哪里调
优化器 = 真正执行调整的人
推理 = 用调好的数字解决新输入
```

对应到这个项目：

```text
stateDict = 那些可调数字
gpt() = 那个函数
loss = 错误程度
backward() = 算每个数字的责任
Adam = 调整数字
runInference() = 用学到的规律生成名字
```

如果你能把这条链路理解清楚，就已经抓住了现代深度学习训练的主干。
